# STT-Diktat (Speech-to-Text, self-hosted)

Sprach-Diktat im **Notebook-Editor** (Einzelseiten-Edit-Modus). Self-hosted, vom Betreiber konfigurier- und abschaltbar. Zweite **Sync-Proxy-Ausnahme** zur Job-Queue-Regel (analog [docs/languagetool.md](languagetool.md)): kurzer Request/Response-Transkriptionscall, **kein** KI-Analysejob, kein Token-Budget, kein `callAI`. STT transkribiert 1:1 — keine generative KI.

## Datenfluss

```
Notebook-Toolbar Mic-Button (Root-Scope) → MediaRecorder + WebAudio-RMS-VAD
  → an Sprechpause: Segment-Blob → POST /stt/transcribe?bookId=…
    → Proxy forwarded an ${stt.host}/v1/audio/transcriptions (OpenAI-kompatibel)
      → { text } → verbatim am Cursor eingefügt (createTextNode/range)
        → normaler Notebook-Save-/Block-ID-Chokepoint (Autosave, data-bid, Stale-Write)
```

## Backend

- **[routes/stt.js](../routes/stt.js)** — `POST /stt/transcribe`. Audio als rohes Binary (`express.raw`, kein multer), 5 MB Cap. Guard: `stt.enabled` + `stt.host` → sonst `404 stt_disabled`. Mime-Whitelist (webm/ogg/m4a/wav/mp3) → korrekte Datei-Extension fuer den ffmpeg-basierten Whisper-Endpunkt; unbekannt → `415`. Forward via `FormData`/`Blob`, optional `Authorization: Bearer`. Timeout 15 s → `408`, Upstream-Fehler → `502`. Audio nie persistiert, nie geloggt (nur Metadaten). `/stt/` steht in `API_PREFIXES` ([server.js](../server.js)) → abgelaufene Session liefert `401 JSON` (greift den globalen Session-Banner-Wrapper) statt eines Login-HTML-Redirects.
- **Sprache:** Buch-Locale gewinnt (`getBookLocale`, SSoT wie LanguageTool); `de-CH` → `de` gekuerzt (Whisper akzeptiert nur ISO-639-1, keine Region). `stt.language` nur Fallback ohne Buchscope.
- **Temperatur:** `stt.temperature` (App-Setting, Default 0) wird pro Call als `temperature` an den Whisper-Endpunkt forwarded. 0 = deterministisch, weniger Halluzinationen bei stillen/unklaren Segmenten. **Bewusst KEIN Eigennamen-`initial_prompt`** (Figuren-/Ortsnamen): ein langer Namens-Prompt verstaerkt Whisper-Halluzinationen statt sie zu unterdruecken.
- **[routes/admin-settings.js](../routes/admin-settings.js)** — `POST /admin/settings/test-stt` (Health-Probe `GET /v1/models`).
- **[routes/proxies.js](../routes/proxies.js)** `/config` — liefert `stt: { enabled, provider, vad: { silenceMs, threshold, maxSegmentS } }`. **Kein** Host/Key/Model/Language (Secret-Leck-Schutz; Sprache loest der Proxy serverseitig auf).
- **[lib/app-settings.js](../lib/app-settings.js)** — Keys: `stt.enabled`, `stt.host`, `stt.model`, `stt.language` (Fallback), `stt.temperature` (0–1), `stt.api_key` (`ENCRYPTED_KEYS`), `stt.vad.silence_ms` (200–5000), `stt.vad.threshold` (0–1), `stt.vad.max_segment_s` (5–120). VALIDATORS-Ranges deckungsgleich mit den `numInput`-Limits im Admin-UI.

## Frontend

- **[public/js/editor/notebook/stt-dictation.js](../public/js/editor/notebook/stt-dictation.js)** — `sttDictationMethods`, in den **Root** gespreaded (die `.page-editor-toolbar` laeuft im Root-Scope `lektorat`, nicht in `editorToolbarCard`). Pure-Compute (`_computeRms`/`_computeVadCut`/`_computeSttMime`/`_computeSpacedInsert`) testbar ohne Browser. Segmentierung via MediaRecorder-**Stop/Start-Zyklus** (stop() liefert standalone-dekodierbares Segment; blosses Slicen ergaebe headerlose Fragmente). `getUserMedia` fordert explizit `echoCancellation`/`noiseSuppression`/`autoGainControl` + `channelCount: 1` (Mono) an — kleinere Segmente, weniger Halluzinationen an der Quelle (Boolean-Constraints best-effort, kein Throw bei fehlendem Support). **Bekannte Limitierung:** ein `max`-Cut (Dauer-Sprechen ohne `silenceMs`-Pause) schneidet mitten im Wort, die Stop/Start-Luecke verliert dabei wenige ms Audio (bei `silence`-Cuts faellt die Luecke in die Pause).
- **Einfüge-Anker (bewusster Caret vs. Editorende):** `_sttStart` entscheidet beim Aufnahmestart — hat der User bewusst per Klick einen Caret im Edit-Feld gesetzt (`sttCaretUserSet`, in [figur-lookup.js](../public/js/editor/figur-lookup.js)#`_onEditClick`) **und** steht dieser noch im Editor (`_sttCaretInEditor`), wird dort eingefuegt (nur Scroll dorthin); sonst — blosser Mic-Klick ohne Caret-Platzierung — `_sttAnchorToEnd()` (Caret ans Editorende + Scroll). Folgesegmente haengen am vorrueckenden Caret weiter an. `sttCaretUserSet` wird in `startEdit` (Auto-Fokus-Caret zaehlt nicht) und bei `book:changed`/`view:reset`/Seitenwechsel auf `false` zurueckgesetzt.
- **Mitscrollen:** Nach jedem Segment-Insert (programmatisch, der Browser zieht den Scroll dabei NICHT automatisch nach) vermisst `_sttInsertText` den eingefuegten Textknoten und ruft `_scrollEditCaretIntoView(rect)` ([notebook/edit.js](../public/js/editor/notebook/edit.js), via Trampoline). Generischer Caret-Follow-Helper: das contenteditable ist sein eigener Scroll-Container (`max-height`/`overflow-y:auto`), darum `scrollTop`-Nudge statt `scrollIntoView`; nur wenn der Caret ueber/unter den Rand rutscht. Auch von `_markEditDirty` aufgerufen (Tippen/Paste/Toolbar — Sicherheitsnetz; No-op solange der Caret sichtbar ist).
- **Grenz-Art (Satz vs. Absatz) an Sprechpausen:** Jedes Segment traegt eine Grenz-Art `'none' | 'sentence' | 'paragraph'` fuer die VORAUSGEHENDE Grenze. Wird per VAD an einer Stille abgeschnitten (`_computeVadCut` reason `silence`, nicht `max`), gilt die naechste Grenze mind. als Satzgrenze; ein `max`-Cut (Dauer-Sprechen) erzeugt keine Grenze. Der Cut-Grund reist via `rt.lastCutReason` → `rt.boundaryKindForNext` zum naechsten `onstop` und von dort durch `_sttSendSegment` → `_sttInsertText`. **Satzgrenzen folgen ausschliesslich der Punktierung des Modells, nicht der VAD-Pause:** `_computeSpacedInsert(prevText, text)` ergaenzt **nie** einen eigenen Punkt — eine blosse Atempause ist kein Satzende, und Whisper punktiert selbst. Erkennt der pure Helper `_endsSentence` am Vortext ein Satzzeichen — auch hinter einer schliessenden Anfuehrung/Klammer (`…her.«`, `?"`, `!)`, strippt schliessende Quotes/Brackets vor dem Test) —, wird nur ein Leerzeichen gesetzt; sonst ebenfalls nur ein Leerzeichen. Dafuer liest `_sttTextBefore` die letzten ~12 Zeichen links vom Caret statt nur ein Zeichen. Das Leerzeichen-vor-Wort-Problem ueber Segmentgrenzen loest `_sttCharBefore` (liest das Zeichen links vom Caret per Range — deckt auch Element-Knoten-Carets nach dem zuletzt eingefuegten Textknoten ab).
- **Absatz-Erkennung (lange Pause):** Beim ersten Sprechen nach einem silence-Cut misst `_sttVadTick` die Gesamtpause (`silenceMs` vor dem Cut + Luecke bis zum neuen Sprechen, via `rt.silenceCutAt`). Ist sie `>= silenceMs * STT_PARAGRAPH_FACTOR` (2.5), wird `rt.boundaryKindForNext` auf `'paragraph'` hochgestuft. `_sttInsertText` delegiert dann an `_sttInsertParagraph`: neuer `<p>` hinter dem Block, in dem der Caret steht (sonst ans Editorende), erster Buchstabe gross, vorausgehender Block bekommt via `_sttEnsureTerminalPunct` ein Satzendezeichen, falls es fehlt. `data-bid` vergibt der Write-Chokepoint beim Speichern.
- **Satzanfang-Grossschreibung:** `_computeSpacedInsert` schreibt den ersten Buchstaben **nur** gross, wenn ein echter Satz beginnt — Doc-/Block-Anfang oder der Vortext endet (per `_endsSentence`) auf einem Satzzeichen. Eine blosse VAD-Sprechpause loest **keine** Grossschreibung aus (sonst wuerde ein durch Atemholen zerschnittener Satz faelschlich grossgeschrieben). Pure Helper `_capitalizeSentenceStart` (beruecksichtigt oeffnende Anfuehrung/Klammer).
- **Halluzinations-Filter:** `_isLikelyHallucination` (pure) verwirft ein Segment, dessen ganzer Text einer bekannten Whisper-Geisterphrase entspricht (Dank-/Abschiedsfloskeln, Untertitel-/Sender-Boilerplate wie ZDF/ARD/Amara/funk, Copyright-Marker). Whole-Segment-Match bzw. eindeutige Marker, damit legitimer Prosatext, der eine solche Floskel *enthaelt*, nicht getilgt wird. Greift in `_sttInsertText` vor Tracking + Insert.
- **VAD-Rauschkalibrierung:** Pro Session sammelt `_sttVadTick` vor dem ersten Sprechen bis zu `STT_CALIB_MS` (350 ms) die ruhigen Frames und hebt `rt.threshold` via `_computeNoiseThreshold(noiseFloor, base)` ueber den Geraeuschboden (nie unter dem Admin-Wert, gedeckelt auf 5× bzw. 0.08). Blockiert die Spracherkennung NICHT (bis zur Finalisierung gilt der Admin-Wert); wird sofort gesprochen oder fehlen ruhige Frames, bleibt es beim Admin-Wert. Der Tick nutzt durchgehend `rt.threshold` statt `sttVad.threshold`.
- **Input-Plausibilisierung:** `_normalizeTranscript` (pure) trimmt jedes Segment, kollabiert interne Whitespace-Folgen (Whisper liefert manchmal Doppel-Leerzeichen/Umbrueche) und tilgt Leerzeichen vor Satzzeichen. Zusaetzlich entfernt `_sttInsertText` via `_computeEatPrevSpace` (pure Entscheidung) + `_sttDeletePrevWhitespace` (DOM) ein bereits vorhandenes Leerzeichen am Caret, wenn das neue Segment mit Satzzeichen beginnt — verhindert „Wort , dann". Greift nur bei kollabiertem Caret.
- **Live-Status-Indikator** ([editor-body-edit.html](../public/partials/editor-body-edit.html), `.stt-dock .stt-status` in [page-view.css](../public/css/page/page-view.css)): Status-Pill links vom Mic mit Punkt-Indikator, **zwei ruhige Zustaende** (bewusst KEIN per-Tick-Pegel-State — der strobte mit jeder Silbe): `stt.status.hearing` (Aufnahme laeuft, roter Punkt mit langsamem Atem-Puls) und `stt.status.transcribing` (Segment-Upload, Akzent-Puls). Der „transkribiert"-Zustand laeuft ueber `sttBusy` mit **Mindest-Standzeit 600 ms** (`_sttBusyOn`/`_sttBusyOff`), damit kurze Segmente nicht aufblitzen. Reduced-Motion: kein Puls. Stoppen = erneuter Klick auf den Mic (`toggleSttDictation`).
- **Segment-Retry:** `_sttFetchTranscript(blob, mime, attempt, signal)` wiederholt transiente Upstream-Fehler (Netzwerk-Throw, Status `408/500/502/503`) bis zu `STT_MAX_RETRY`-mal mit `STT_RETRY_DELAY_MS` Verzoegerung, bevor der Fehler-Toast (`_sttToastFailed`) kommt — ein kurzer Haenger kostet keinen Satz. `404` (Feature aus) und `401` (Session abgelaufen) → `_sttStop` ohne Fehler-Toast (beim 401 kommt der Session-Banner global); andere `4xx` (`413/415`) werden NICHT wiederholt. Ein `200` mit kaputtem Body (Server-/Proxy-Fehler) wird nicht stumm verworfen, sondern loest `_sttToastFailed` aus (ausser bei Abort waehrend des Body-Reads). Der „transkribiert"-Indikator (`_sttBusyOn`/`Off`) umschliesst alle Retries. Das `signal` ist der Session-AbortController (`_sttRt.abort`): `_sttStop` bricht damit Request **und** Retry-Wait sofort ab — ein abgebrochenes Segment liefert `null` ohne Fehler-Toast (kein Insert nach dem Stop).
- **Einfüge-Reihenfolge (parallel transkribieren, seriell einfügen):** Segmente werden per MediaRecorder-Stop/Start fortlaufend gesendet; mehrere Transkriptions-Requests koennen gleichzeitig laufen. `_sttSendSegment` startet den Fetch sofort, haengt die DOM-**Einfuegung** aber an `_sttRt.insertChain` (Promise-Kette in Sende-Reihenfolge). So landet ein frueher gesprochenes Segment auch dann vor einem spaeteren im Text, wenn dessen Transkript — z. B. nach einem Retry oder bei schwankender Upstream-Latenz — erst spaeter zurueckkommt. Der Guard `this._sttRt === rt` verwirft Inserts, deren Session inzwischen beendet/gewechselt wurde (Stop, Seitenwechsel) — zweite Verteidigungslinie zum Abort.
- **State** ([app-state.js](../public/js/app/app-state.js)): `sttEnabled`, `sttVad`, `sttRecording`, `sttPending` (Re-Entry-Guard), `sttTranscribing` (Zaehler laufender Transkriptions-Requests, in `_sttSendSegment`; in `_sttStop` auf 0 zurueckgesetzt), `sttBusy` (abgeleiteter Anzeige-Flag mit Mindest-Standzeit), `sttCaretUserSet` (bewusster Einfuege-Anker). Runtime-Handle `_sttBusyTimer`. Sprache **nicht** im Frontend. Session-Runtime in `_sttRt` (u. a. `threshold`, `calibrating`, `boundaryKindForNext`, `silenceCutAt`, `abort` (AbortController der Session), `insertChain` (Reihenfolge-Serialisierung)).
- **Config-Load + init-Hook** ([app.js](../public/js/app.js)): `this.sttEnabled = !!cfg.stt?.enabled`; `_initSttDictation(signal)` abonniert `book:changed`/`view:reset` und watcht `editMode`/`currentPage.id` → Aufnahme stoppen + Mic freigeben.
- **Button** ([editor-body-edit.html](../public/partials/editor-body-edit.html)): `x-if="sttEnabled"`, schwebender Mic-Button `.stt-dock-btn` **unten rechts im Editorfeld** (`.stt-dock`, `position: sticky`, analog zum LanguageTool-Badge oben rechts — bewusst nicht in der Toolbar, sonst verwechselbar). Icon `#mic`, Recording-State (roter Puls) via CSS in [page-view.css](../public/css/page/page-view.css) (`.stt-dock-btn.is-recording[aria-pressed="true"]`). Sticky haelt den Mic am sichtbaren Feldrand, waehrend Seite/Karte scrollt.
- **Diktat-Tracking** ([stt-time.js](../public/js/book/stt-time.js), Methoden in den Root gespreadet): Heartbeat alle 15 s, solange `sttRecording` aktiv und der Tab sichtbar ist (`_setupSttTime` watcht `sttRecording`/`selectedBookId`, plus `visibilitychange`/`pagehide`-Beacon — analog [writing-time.js](../public/js/book/writing-time.js)). Bucht **Sekunden** (Mic-Wallclock) **und diktierte Zeichen** pro (User, Buch, Tag) buchweit via `POST /history/stt-time` (UPSERT, Server-Clamp 1 h / 100k chars). Die Zeichen liefert `_trackSttChars(n)`, aufgerufen aus [stt-dictation.js](../public/js/editor/notebook/stt-dictation.js)#`_sttInsertText` **nach** Empty- und Halluzinations-Filter (zählt nur tatsächlich eingefügten Text). Auswertung in der Buchentwicklung (BookStats-Karte, `GET /history/stt-time/:book_id` → Tagesreihe `daily[{date,seconds,chars}]`): Chart-Metriken `stt_minutes` / `stt_cumulative` / `stt_chars` ([bookstats.js](../public/js/book/bookstats.js)) + Summen-Badges (`bookstats.sttTotal`/`bookstats.sttChars`). Metrics-Export: `sw_stt_seconds_today` / `sw_stt_chars_today` ([lib/metrics-collector.js](../lib/metrics-collector.js), siehe [metrics-api.md](metrics-api.md)). DB: Tabelle `stt_time` (Migration 173, FK `book_id`→`books`/`user_email`→`app_users`, beide CASCADE).
- **Admin-UI** ([admin-settings.html](../public/partials/admin-settings.html)): Tab `stt` (enabled, Host, Model, API-Key-Masking, Fallback-Sprache-`combobox`, VAD-`numInput`s, Test-Button). Save/Diff/Encrypted-Coercion generisch ueber das bestehende `admin-settings.js`-Geruest.

## Pflicht-Invarianten

- **Nur Notebook-Editor.** Focus-Editor + Bucheditor unberuehrt (Phase 2).
- Eingefuegter Text laeuft durch den normalen Notebook-Save-Pfad — STT ist nur eine weitere DOM-Mutation am contenteditable, danach greift Autosave/`data-bid`/Stale-Write unveraendert.
- Transkript als **reiner Text** (`createTextNode`/`range.insertNode`) — keine HTML-Interpretation, kein `x-html`-Sink.
- Host/API-Key bleiben server-seitig; `/config` liefert sie nie.
- Kein Audio persistiert. Leerer/Whitespace-Transkript → nichts einfuegen. Einzelner Segment-Fehler stoppt die Session nicht.
- **Reihenfolge bleibt Sprechreihenfolge** — parallele Transkripte werden in Sende-Reihenfolge eingefuegt (`insertChain`), nie nach Aufloese-Reihenfolge.
- **Kein Insert nach Stop** — `_sttStop` bricht laufende Requests/Retries ab (`_sttRt.abort`); zusaetzlich verwirft der Session-Guard (`this._sttRt === rt`) spaet eintreffende Transkripte.
- Abschaltbar ueber `stt.enabled=false` (Default) → Button nicht im DOM, Proxy `404`.
