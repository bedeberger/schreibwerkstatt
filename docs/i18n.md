# i18n

Code: [public/js/i18n.js](../public/js/i18n.js), Strings: [public/js/i18n/de.json](../public/js/i18n/de.json) + [public/js/i18n/en.json](../public/js/i18n/en.json).

## Regel

**Jeder User-sichtbare String ist i18n-Key — Frontend, Server, Fehler, Status, Tooltip, Placeholder.** Hartcodierter DE/EN-Text in HTML/JS/Alpine-Templates ist verboten.

`de` ist Fallback-Locale. `en` ist Übersetzung. Neuer String → Eintrag in **beiden** Dateien, im selben Commit. Kein "mach ich später".

Ausnahme: Winston-Logs (`logger.info/warn/error`) bleiben deutsch — gehen nur in `schreibwerkstatt.log`/Console, nicht an User.

## Key-Konvention

`bereich.feld` — z.B. `profile.title`, `job.phase.aiReply`, `error.OLLAMA_UNREACHABLE`.

Platzhalter: `{name}` im JSON, Parameter-Map beim Call.

```js
t('lektorat.batchDone', { done: 5, total: 12, errors: 3 });
// "Fertig: 5/12 Seiten geprüft, 3 Beanstandungen."
```

## Frontend-API

```js
import { tRaw, configureI18n } from './i18n.js';
```

| Funktion | Wo | Verhalten |
|----------|-----|-----------|
| `t(key, params?)` | Alpine-Methode (in Templates: `$app.t(...)`) | Reaktiv ueber `this.uiLocale` — re-rendert bei Locale-Switch |
| `tRaw(key, params?)` | Pure JS (ausserhalb Alpine) | Nicht reaktiv |
| `tError(response)` | Alpine, Backend-Fehler | Akzeptiert `{ error_code, params }` oder `{ error: '…' }` |
| `tErrorRaw(response)` | Pure JS | Nicht reaktiv |
| `changeLocale(locale)` | Alpine | Laedt Messages, persistiert via `PATCH /me/settings` |

Resolve-Reihenfolge: aktive Locale → Fallback (`de`) → Key selbst (sichtbares Debug-Signal).

## Server-Side: Job-Status & Fehler

`updateJob`/`failJob` in [routes/jobs/shared/jobs.js](../routes/jobs/shared/jobs.js) erwarten i18n-Keys.

```js
updateJob(jobId, {
  statusText: 'job.phase.processingChapter',
  statusParams: { i: 3, n: 12 },
  progress: 45,
});
```

`updateJob({ statusText: ... })` ohne `statusParams` setzt `statusParams=null` automatisch — Platzhalter aus frueherer Phase wirken nicht nach.

**Job-Labels** an `createJob`: `label` als String-Key, `labelParams` als Map.

**Fehler**:

```js
const { i18nError } = require('./shared');
throw i18nError('error.CHAPTER_NOT_FOUND', { chapterId });
```

`failJob` extrahiert `err.i18nParams` als `errorParams`. Frontend rendert `t(job.error, job.errorParams)`.

Sentinel: `'job.cancelled'` (gesetzt automatisch bei AbortError).

## Persistierte Nachrichten in DB (Chat-Fallbacks)

Wenn Server eine Nachricht in DB speichert, die spaeter beim Lesen lokalisiert werden muss: Marker `__i18n:bereich.feld__` als Wert speichern.

```js
// Server-Side persist
finalText = JSON.stringify({ antwort: '__i18n:chat.errors.contextExceeded__' });
```

Frontend loest beim Rendern auf:

```js
// public/js/chat/chat-base.js
const match = /^__i18n:([a-zA-Z0-9_.-]+)__$/.exec(text || '');
if (match) text = t(match[1]);
```

Vorteil: Locale-Wahl des **spaeteren Betrachters** ist massgeblich, nicht die zur Zeit des Schreibens.

## Backend-Fehler-Schema

Server-JSON-Antworten mit Fehler:

```json
{ "error_code": "OLLAMA_UNREACHABLE", "params": { "host": "localhost:11434" } }
```

Frontend: `tError(response)` → `t('error.OLLAMA_UNREACHABLE', { host: 'localhost:11434' })`.

Legacy-Fallback `{ error: 'freier Text' }` wird durchgereicht (vermeiden, nur fuer nicht migrierten Code).

## Locale-Wechsel

`changeLocale(locale)` aus `i18nMethods`:
1. Laedt `/js/i18n/<locale>.json`.
2. Setzt `this.uiLocale = locale` (Alpine triggert Re-Render aller `t()`-Calls).
3. `document.documentElement.lang = locale`.
4. `PATCH /me/settings { locale }` persistiert serverseitig.

Beim Login laedt `configureI18n(user.locale)` direkt die User-Locale + Fallback.

## Neuer String — Checkliste

1. Key erfinden (`bereich.feld`).
2. Eintrag in [de.json](../public/js/i18n/de.json) + [en.json](../public/js/i18n/en.json).
3. Im Template: `t('bereich.feld')` (Alpine) oder `tRaw(...)` (Pure JS).
4. Bei Server-Status/Fehler: `statusText`/`i18nError` mit Key.
5. Bei DB-persistierter Nachricht: `__i18n:bereich.feld__`-Marker.
